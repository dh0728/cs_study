# 정렬 알고리즘(Sorting Algorithm)

> 정렬 알고리즘(Sorting Algorithm)은 **데이터를 특정 기준(오름차순/내림차순)에 따라 순서대로 재배열하는 알고리즘**

---

## 정렬 알고리즘을 보는 기준

정렬 알고리즘은 시간복잡도뿐 아니라 다음 세 가지 성질로도 구분한다.

- **비교 정렬(Comparison Sort):** 두 요소를 비교하여 순서를 결정하는 정렬. 버블·선택·삽입·퀵·합병·힙 정렬 모두 비교 정렬에 속한다.
- **안정 정렬(Stable Sort):** 값이 같은 요소들의 **원래 순서가 정렬 후에도 유지**되는 정렬
- **제자리 정렬(In-place Sort):** 입력 배열 외에 **추가 메모리를 거의 사용하지 않는($O(1)$)** 정렬

```
안정 정렬 예시 (같은 값 3의 순서 유지)

입력:  3a  1  3b  2
정렬:  1  2  3a  3b   ← 3a가 3b보다 앞에 유지됨 (안정)
정렬:  1  2  3b  3a   ← 순서가 뒤바뀜 (불안정)
```

---

## 버블 정렬 (Bubble Sort)

- 인접한 두 요소를 비교하여 큰 값을 뒤로 보내는 방식
- 한 번 순회할 때마다 가장 큰 값이 거품처럼 맨 뒤로 떠오른다
- 가장 단순하지만 가장 느린 정렬 중 하나

### 동작 과정

```
[5, 3, 8, 1]  첫 번째 순회

5 > 3 → swap   [3, 5, 8, 1]
5 < 8 → 유지   [3, 5, 8, 1]
8 > 1 → swap   [3, 5, 1, 8]   ← 가장 큰 8이 맨 뒤 확정

[3, 5, 1, 8]  두 번째 순회
3 < 5 → 유지   [3, 5, 1, 8]
5 > 1 → swap   [3, 1, 5, 8]   ← 5 확정

[3, 1, 5, 8]  세 번째 순회
3 > 1 → swap   [1, 3, 5, 8]   ← 완료
```

### 슈도코드

```
bubbleSort(arr):
    n = length(arr)
    for i from 0 to n-1:
        swapped = false
        for j from 0 to n-i-2:
            if arr[j] > arr[j+1]:
                swap(arr[j], arr[j+1])
                swapped = true
        if swapped == false:    # 교환이 없으면 이미 정렬됨
            break
```

### 시간복잡도

- **최선:** $O(n)$ (이미 정렬된 경우, swapped 최적화 시)
- **평균 / 최악:** $O(n^2)$
- **공간:** $O(1)$ · **안정 정렬** · **제자리 정렬**

## 선택 정렬 (Selection Sort)

- 전체에서 **최솟값을 찾아** 맨 앞 요소와 교환하는 방식
- 정렬된 영역을 앞에서부터 하나씩 늘려나간다
- 데이터 상태와 무관하게 항상 같은 횟수를 비교한다

### 동작 과정

```
[5, 3, 8, 1]

최솟값 1 찾음 → 맨 앞(5)과 swap   [1, 3, 8, 5]
나머지 중 최솟값 3 → 제자리       [1, 3, 8, 5]
나머지 중 최솟값 5 → 8과 swap     [1, 3, 5, 8]   ← 완료
```

### 슈도코드

```
selectionSort(arr):
    n = length(arr)
    for i from 0 to n-1:
        min_idx = i
        for j from i+1 to n-1:
            if arr[j] < arr[min_idx]:
                min_idx = j
        swap(arr[i], arr[min_idx])
```

### 시간복잡도

- **최선 / 평균 / 최악:** $O(n^2)$ (항상 전체를 비교)
- **공간:** $O(1)$ · **불안정 정렬** · **제자리 정렬**

> 💡 선택 정렬은 비교는 $O(n^2)$로 많지만 **교환(swap)은 최대 $O(n)$번**만 발생한다. 교환 비용이 큰 환경에서는 유리할 수 있다.

## 삽입 정렬 (Insertion Sort)

- 현재 요소를 **앞쪽 정렬된 부분의 알맞은 위치에 삽입**하는 방식
- 카드를 한 장씩 뽑아 손 안의 정렬된 카드 사이에 끼워넣는 것과 같다
- 거의 정렬된 데이터에서 매우 빠르다

### 동작 과정

```
[5, 3, 8, 1]

3 삽입 → 5보다 작으므로 앞으로    [3, 5, 8, 1]
8 삽입 → 5보다 크므로 제자리      [3, 5, 8, 1]
1 삽입 → 맨 앞까지 이동           [1, 3, 5, 8]   ← 완료
```

### 슈도코드

```
insertionSort(arr):
    for i from 1 to n-1:
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j+1] = arr[j]    # 한 칸씩 뒤로 밀기
            j = j - 1
        arr[j+1] = key           # 빈 자리에 삽입
```

### 시간복잡도

- **최선:** $O(n)$ (이미 정렬된 경우)
- **평균 / 최악:** $O(n^2)$
- **공간:** $O(1)$ · **안정 정렬** · **제자리 정렬**

> 💡 데이터가 거의 정렬되어 있을수록 빠르기 때문에, 작은 부분 배열 정렬에 자주 활용된다. (퀵 정렬·합병 정렬에서 작은 구간을 삽입 정렬로 처리하는 하이브리드 방식이 있다)

## 퀵 정렬 (Quick Sort)

- **피벗(pivot)** 을 기준으로 작은 값과 큰 값으로 나누고, 각 부분을 재귀적으로 정렬
- **분할 정복(Divide and Conquer)** 방식
- 평균적으로 가장 빠른 정렬로 실무에서 널리 쓰인다

### 동작 과정

```
[5, 3, 8, 1, 4]   pivot = 4 (맨 끝)

피벗 4보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 분할
        [3, 1] [4] [5, 8]
          ↓          ↓
      재귀 정렬    재귀 정렬
        [1, 3]     [5, 8]

결과: [1, 3, 4, 5, 8]
```

### 슈도코드

```
quickSort(arr, low, high):
    if low < high:
        p = partition(arr, low, high)   # 피벗 위치 확정
        quickSort(arr, low, p - 1)      # 왼쪽 부분
        quickSort(arr, p + 1, high)     # 오른쪽 부분

partition(arr, low, high):
    pivot = arr[high]
    i = low - 1
    for j from low to high-1:
        if arr[j] < pivot:
            i = i + 1
            swap(arr[i], arr[j])
    swap(arr[i+1], arr[high])           # 피벗을 가운데로
    return i + 1
```

### 시간복잡도

- **최선 / 평균:** $O(n \log n)$
- **최악:** $O(n^2)$ (피벗이 항상 한쪽 끝값으로 치우칠 때)
- **공간:** $O(\log n)$ (재귀 호출 스택) · **불안정 정렬** · **제자리 정렬**

> 💡 **최악을 피하는 방법**
> 정렬된 배열에서 맨 끝을 피벗으로 잡으면 분할이 한쪽으로 쏠려 $O(n^2)$이 된다. 피벗을 무작위로 고르거나(랜덤 피벗), 세 값의 중앙값(median-of-three)을 쓰면 최악을 거의 피할 수 있다.

## 합병 정렬 (Merge Sort)

- 배열을 **절반으로 계속 나눈 뒤**, 정렬하면서 **병합(merge)** 하는 방식
- **분할 정복** 방식
- 데이터 상태와 무관하게 항상 $O(n \log n)$을 보장한다

### 동작 과정

```
분할 (계속 절반으로)
        [5, 3, 8, 1]
         /         \
     [5, 3]       [8, 1]
      / \          / \
    [5] [3]      [8] [1]

병합 (정렬하면서 합치기)
    [3, 5]       [1, 8]
         \         /
        [1, 3, 5, 8]   ← 완료
```

### 슈도코드

```
mergeSort(arr):
    if length(arr) <= 1:
        return arr
    mid  = length(arr) / 2
    left  = mergeSort(arr[0 : mid])
    right = mergeSort(arr[mid : ])
    return merge(left, right)

merge(left, right):
    result = []
    while left and right are not empty:
        if left[0] <= right[0]:
            result.append(remove_first(left))
        else:
            result.append(remove_first(right))
    result.append(남은 요소 전부)
    return result
```

### 시간복잡도

- **최선 / 평균 / 최악:** $O(n \log n)$ (항상 동일)
- **공간:** $O(n)$ (병합용 추가 배열 필요) · **안정 정렬** · **제자리 아님**

> 💡 분할은 $\log n$ 단계로 진행되고, 각 단계마다 전체 $n$개를 병합하므로 $O(n \log n)$이다. 추가 메모리를 쓰는 대신 **항상 안정적인 성능**을 보장한다.

## 힙 정렬 (Heap Sort)

- **최대 힙**을 구성한 뒤, 루트(최댓값)를 꺼내 맨 뒤로 보내는 과정을 반복하는 방식
- 힙의 성질(루트가 항상 최댓값)을 이용한다
- 추가 메모리 없이 $O(n \log n)$을 보장한다

### 동작 과정

```
1. 배열로 최대 힙 구성 (Build Heap)
        9
       / \
      5   8
     / \
    1   3

2. 루트(9)를 맨 뒤로 보내고 힙 크기 축소 → heapify
   [_, 5, 8, 1, 3 | 9]

3. 다시 루트(8)를 맨 뒤로 → 반복
   [_, 5, 3, 1 | 8, 9]

   ... 반복하면 정렬 완료 → [1, 3, 5, 8, 9]
```
```
 배열  :[9, 5, 8, 1, 3]
 인덱스: 0  1  2  3  4

i=4 → 루트를 인덱스 4(맨 뒤)로 보냄   [_, _, _, _ | 10]
i=3 → 루트를 인덱스 3으로 보냄        [_, _, _ | 8, 10]
i=2 → 루트를 인덱스 2로 보냄          [_, _ | 5, 8, 10]
i=1 → 루트를 인덱스 1로 보냄          [_ | 3, 5, 8, 10]

남은 1은 자동으로 맨앞
```

### 슈도코드

```
heapSort(arr):
    buildMaxHeap(arr)              # 전체를 최대 힙으로 → O(n)
    for i from n-1 down to 1:
        swap(arr[0], arr[i])       # 루트(최댓값)를 맨 뒤로
        heapify(arr, 0, i)         # 줄어든 힙 다시 정렬 → O(log n)

heapify(arr, root, size):          # sift-down
    largest = root
    left  = 2*root + 1
    right = 2*root + 2
    if left < size and arr[left] > arr[largest]:
        largest = left
    if right < size and arr[right] > arr[largest]:
        largest = right
    if largest != root:
        swap(arr[root], arr[largest])
        heapify(arr, largest, size)
```

### 시간복잡도

- **최선 / 평균 / 최악:** $O(n \log n)$ (항상 동일)
- **공간:** $O(1)$ · **불안정 정렬** · **제자리 정렬**

> 💡 힙 생성(Build Heap)은 $O(n)$, 이후 $n$번의 루트 추출이 각각 $O(\log n)$이므로 전체 $O(n \log n)$이다. 합병 정렬과 달리 **추가 메모리가 필요 없다**.

### 한눈에 보는 선택 기준

- **단순 구현·학습용:** 버블 / 선택 / 삽입 정렬
- **거의 정렬된 데이터:** 삽입 정렬 ($O(n)$에 근접)
- **평균 성능이 가장 중요:** 퀵 정렬
- **최악에도 안정적인 성능 보장:** 합병 정렬, 힙 정렬
- **추가 메모리를 못 쓰는 환경:** 힙 정렬
