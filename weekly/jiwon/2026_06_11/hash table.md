# 해시 테이블 (Hash Table)

## 해시 테이블이란?

**키를 해시 함수로 변환한 인덱스**에 값을 저장하는 자료구조다. 크기가 제각각인 데이터들을 고정된 인덱스 범위로 매핑해서 일정한 속도로 접근하기 위해 사용된다. 맵과 셋의 내부 구현에 사용된다.

```
키 "Alice"
    ↓ 해시 함수
   인덱스 3
    ↓
배열[3] = "Alice"
```

---

## 해시 함수 (Hash Function)

임의의 키를 고정된 범위의 정수 인덱스로 변환하는 함수다.

```python
def hash_function(key, size):
    return sum(ord(c) for c in key) % size

hash_function("Alice", 10)   # → 특정 인덱스
```

좋은 해시 함수는 키를 고르게 분산시켜 충돌을 최소화한다.

---

## 해시 충돌 (Hash Collision)

서로 다른 키가 같은 인덱스로 변환되는 경우다.

```
"Alice"  →  인덱스 3
"Bob"    →  인덱스 3   ← 충돌 발생
```

### 충돌 해결 방법

**체이닝 (Chaining)**: 같은 인덱스에 연결리스트로 여러 값을 저장한다.

```
배열[3] → ["Alice" | →] → ["Bob" | None]
```

**개방 주소법 (Open Addressing)**: 충돌 발생 시 빈 슬롯을 찾아 저장한다. 빈 슬롯을 찾는 방식에 따라 세 가지로 나뉜다.
 
- **선형 탐사 (Linear Probing)**: 충돌 시 다음 버켓, 혹은 몇 개를 건너뛰어 삽입한다.
```
충돌 → +1 → +1 → +1 ...
배열[3] = "Alice"
배열[4] = "Bob"    ← 3이 차있으므로 다음 슬롯에 저장
```
 
- **제곱 탐사 (Quadratic Probing)**: 충돌 시 제곱만큼 건너뛴 버켓에 삽입한다.
```
충돌 → +1 → +4 → +9 → +16 ...
```
 
- **이중 해시 (Double Hashing)**: 충돌 시 다른 해시 함수를 한 번 더 적용한 결과를 간격으로 사용한다.
```
충돌 → +hash2(key) → +hash2(key) ...
```

---

## 시간복잡도

| 연산 | 평균 | 최악 (충돌 많을 때) |
|------|------|---------------------|
| 삽입 | O(1) | O(n) |
| 삭제 | O(1) | O(n) |
| 조회 | O(1) | O(n) |

충돌이 적을수록 평균 O(1)에 가까워진다.

---

## 맵/셋과의 관계

```
파이썬 dict  →  해시 테이블로 구현된 맵
파이썬 set   →  해시 테이블로 구현된 셋
```

맵과 셋에서 조회가 O(1)인 이유가 내부적으로 해시 테이블을 사용하기 때문이다.

---

## 정리

- 해시 테이블은 해시 함수로 키를 인덱스로 변환해 값을 저장하는 자료구조다
- 평균 O(1)의 삽입/삭제/조회가 가능하다
- 서로 다른 키가 같은 인덱스를 가리키는 충돌이 발생할 수 있다
- 충돌 해결 방법으로 체이닝과 개방 주소법이 있다
- 파이썬의 `dict`와 `set`이 해시 테이블로 구현되어 있다
