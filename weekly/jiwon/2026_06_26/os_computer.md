## 운영체제란

운영체제는 하드웨어와 사용자 사이에서 다음 두 가지 역할을 한다.

- **자원 할당자(Resource Allocator):** CPU, 메모리, 입출력 장치 등 한정된 자원을 여러 프로그램에 공평하고 효율적으로 분배한다.
- **제어 프로그램(Control Program):** 프로그램 실행을 통제하고, 오류나 부적절한 자원 사용을 막는다.

### 운영체제의 목표

1. 사용자 프로그램을 실행시켜 사용자의 문제를 쉽게 해결한다
2. 컴퓨터 시스템을 효율적이고 편리하게 사용할 수 있도록 한다

> 💡 **커널(Kernel)** 은 컴퓨터를 켜는 순간부터 항상 메모리에 상주하며 동작하는 핵심 프로그램이다. 좁은 의미의 운영체제는 곧 커널을 가리킨다.

```
[ 이미지 자리 ]
설명: 사용자 → 응용 프로그램 → 운영체제(커널) → 하드웨어로 이어지는
4계층 구조를 동심원 또는 층층 구조로 표현한 그림
```

## 컴퓨터 시스템의 구조

컴퓨터 시스템은 네 개의 계층으로 구성된다.

| 계층 | 설명 |
| --- | --- |
| **하드웨어** | CPU, 메모리, 입출력 장치 등 물리적인 모든 것 |
| **운영체제** | 하드웨어를 효율적으로 동작시키는 소프트웨어 |
| **응용 프로그램** | 워드, 브라우저 등 각종 응용 소프트웨어 |
| **사용자** | 사람, 다른 기계, 다른 컴퓨터 등 |

### 부팅 (Computer Startup)

컴퓨터를 켜면 가장 먼저 **부트스트랩 프로그램(bootstrap program)** 이 실행된다.

- 일반적으로 ROM이나 EEPROM에 저장된 **펌웨어(firmware)** 형태로 존재한다
- 시스템을 초기화한 뒤, 커널을 메모리에 로딩하고 실행시킨다

```
전원 ON → 부트스트랩 실행 → 시스템 초기화 → 커널 로딩 → 운영체제 동작
```

## 컴퓨터의 요소

컴퓨터 시스템은 하나 이상의 CPU와 여러 장치 컨트롤러로 구성되며, **공용 버스(common bus)** 를 통해 연결된다.

- 메모리는 **word 단위**로 주소가 할당되고, 해당 주소로 접근한다
- 각 하드웨어는 자신의 **컨트롤러(controller)** 를 통해 버스에 연결된다
- CPU와 입출력 장치는 **동시에 실행**될 수 있다

```
        ┌──────┐   ┌──────┐   ┌──────────┐
        │ CPU  │   │ 메모리 │   │ I/O 컨트롤러 │
        └──┬───┘   └──┬───┘   └────┬─────┘
           │          │            │
        ═══╧══════════╧════════════╧═══  ← 공용 버스 (Bus)
```

> 💡 입출력 장치의 컨트롤러에는 **버퍼(buffer)** 가 있다. CPU는 메모리에서 버퍼까지만 데이터를 보내면 출력이 끝난 것으로 간주하고 다음 작업을 진행한다. 버퍼에서 실제 장치로 데이터를 옮기는 것은 입출력 장치가 담당한다.

```
[ 이미지 자리 ]
설명: CPU, 메모리, 여러 개의 장치 컨트롤러(디스크, USB, 그래픽 등)가
하나의 시스템 버스에 연결된 구조도
```

## 인터럽트

**인터럽트(Interrupt)** 는 입출력 작업의 완료 등 특정 사건이 발생했음을 CPU에게 알리는 신호다. "방해"라는 의미 그대로, CPU가 하던 일을 잠시 멈추게 한다.

### 인터럽트 처리 과정

1. 인터럽트가 발생하면 CPU는 **현재까지의 작업 상태와 주소를 저장**한다
   - 다음에 실행할 명령어의 주소는 **프로그램 카운터(PC, Program Counter)** 레지스터에 저장된다
2. 인터럽트에 해당하는 처리 루틴으로 제어권을 넘긴다
   - 어떤 루틴을 실행할지는 **인터럽트 벡터(interrupt vector)** 에 저장되어 있다
3. 인터럽트 처리가 끝나면, 저장해둔 주소로 복귀해 원래 작업을 이어간다

```
명령어 실행 중 ──→ 인터럽트 발생 ──→ 상태/주소 저장
                                        │
                                        ▼
원래 작업 복귀 ←── 주소 복원 ←── 인터럽트 처리 루틴 실행
```

> 💡 인터럽트 처리 도중 또 다른 인터럽트가 발생하면 **우선순위**를 따져 더 급한 것을 먼저 처리한다. 또한 0으로 나누기처럼 발생하면 안 되는 상황은 **트랩(trap)** 으로 처리해 막는다.

```
[ 이미지 자리 ]
설명: CPU가 명령어를 실행하다가 인터럽트가 발생해 처리 루틴으로 갔다가
다시 원래 위치로 복귀하는 타임라인(instruction cycle → interrupt cycle) 그림
```

## 입출력 구조와 DMA

입출력 방식은 크게 동기식과 비동기식으로 나뉜다.

| 방식 | 설명 | 동시 작업 |
| --- | --- | --- |
| **동기식 입출력** | CPU가 입출력을 요청한 뒤 끝날 때까지 함께 기다림 | 최대 1개 |
| **비동기식 입출력** | CPU가 입출력을 요청하고 다른 작업을 진행 | 여러 개 가능 |

비동기식에서는 여러 입출력 장치의 상태를 **device-status table** 로 관리한다.

### DMA (Direct Memory Access)

입출력 장치 컨트롤러에 **DMA 칩**을 탑재하면, CPU를 거치지 않고 데이터를 직접 메모리와 주고받을 수 있다.

- DMA 컨트롤러가 데이터를 **블록(block) 단위**로 처리한다
- 매번 인터럽트를 보내지 않고, **블록 처리가 끝났을 때 한 번만** 인터럽트를 발생시킨다
- CPU의 개입을 줄여 효율적이다

```
일반 입출력:  바이트마다 → 인터럽트 → CPU 개입 (잦은 방해)
DMA:         블록 전체 처리 후 → 인터럽트 1회 (CPU 부담 ↓)
```

## 듀얼 모드와 시스템 콜

운영체제는 사용자 프로그램이 하드웨어에 함부로 접근하지 못하도록 **두 가지 실행 모드**를 둔다.

### 듀얼 모드 (Dual Mode)와 모드 비트 (Mode Bit)

| 모드 | mode bit | 설명 |
| --- | --- | --- |
| **사용자 모드 (User Mode)** | 1 | 일반 응용 프로그램 실행. 제한된 권한 |
| **커널 모드 (Kernel Mode)** | 0 | 운영체제가 하드웨어를 직접 제어. 모든 권한 |

mode bit 하나로 현재 어떤 모드인지 구분한다. 사용자 프로그램이 하드웨어 제어 같은 특권 작업을 하려면 운영체제에 요청해야 한다.

### 시스템 콜 (System Call)

사용자 프로그램이 운영체제의 기능을 사용하기 위해 호출하는 것이 **시스템 콜**이다.

```
사용자 프로그램이 특권 작업 필요
        ↓
시스템 콜 호출 → trap 발생 → mode bit = 0 (커널 모드 전환)
        ↓
운영체제가 작업 수행
        ↓
작업 완료 → mode bit = 1 (사용자 모드 복귀)
```

- 사용자는 보통 **API(Application Programming Interface)** 를 통해 간접적으로 시스템 콜을 사용한다
- 대표적 API: Win32(Windows), POSIX(UNIX/Linux/iOS), Java API(JVM)

> 💡 예를 들어 C 코드에서 `printf()`를 호출하면, 내부적으로 `write()` 라는 시스템 콜이 실행된다. 사용자는 시스템 콜의 존재를 몰라도 API만 알면 된다.

```
[ 이미지 자리 ]
설명: 사용자 모드에서 시스템 콜 호출 → trap → 커널 모드 전환 →
작업 수행 → 사용자 모드 복귀로 이어지는 모드 전환 다이어그램
```
