# [CS 스터디] 컴퓨터 구조 및 운영체제 핵심 정리

## 💻 Part 1. 운영체제와 컴퓨터

---

### 1. 운영체제의 역할과 구조

**운영체제(OS, Operating System)**는 하드웨어라는 기계 덩어리에 생명을 불어넣고, 사용자가 소프트웨어를 편리하게 실행할 수 있도록 관리해 주는 '최고 관리자'입니다. 컴퓨터를 켜면 하드웨어 위에 가장 먼저 OS가 자리를 잡아야 비로소 우리가 아는 컴퓨터가 작동합니다.

#### 🛡️ 운영체제의 4대 핵심 역할
1. **프로세스 및 스레드 관리**: 어떤 프로그램에 CPU를 언제, 얼마나 줄지 결정 (스케줄링).
2. **메모리 관리**: 어떤 프로그램에 어느 정도의 RAM 공간을 할당하고 회수할지 통제.
3. **파일 시스템 관리**: 디렉터리 구조를 만들고 파일을 저장, 읽기, 쓰기 관리.
4. **I/O(입출력) 및 주변장치 관리**: 모니터, 키보드, 마우스, 네트워크 카드 등의 드라이버를 제어.
---
#### 🏗️ 운영체제의 구조 (커널과 인터페이스)

[ 사용자 (User) / 응용 프로그램 (Apps) ]
│
[ 인터페이스 ] (GUI / CLI)

★ [ 시스템 콜 (System Call) ] ★
│
[ 커널 (Kernel) ]
│
[ 하드웨어 (Hardware) ]

* **커널 (Kernel)**: 운영체제의 핵심 중의 핵심. 하드웨어를 직접 제어하고 자원을 분배하는 소스코드가 집약된 곳입니다. 보안과 시스템 안정성이 매우 중요하므로 아무나 접근할 수 없도록 보호됩니다.
* **인터페이스 (Interface)**: 사용자가 커널에 명령을 내릴 수 있게 해주는 껍데기입니다. 그래픽 위주인 **GUI**(Windows 창, macOS 바탕화면 등)와 텍스트 위주인 **CLI**(터미널, CMD)가 있습니다.
* **시스템 콜 (System Call)**: 응용 프로그램이 프린터를 쓰거나 파일을 저장하는 등 하드웨어 자원이 필요할 때, **커널에 "이것 좀 대신 해줘"라고 안전하게 요청하는 통로(API)**입니다. 유저 모드에서 커널 모드로의 전환이 일어납니다.

### 2. 컴퓨터의 요소

컴퓨터의 하드웨어는 크게 CPU, 메모리, 그리고 주변 장치들이 버스(Bus)로 연결된 구조입니다. 프로그램을 메모리에 올려서 순서대로 실행하는 이 방식을 **폰 노이만 구조**라고 부릅니다.

#### 🎯 CPU (중앙처리장치)의 3대 구성 요소
CPU는 컴퓨터의 두뇌이며, 내부적으로 크게 세 가지 방으로 나뉩니다.
1. **ALU (산술논리연산장치)**: 실제 덧셈, 뺄셈 같은 수학 연산과 AND, OR 같은 논리 연산을 수행하는 '계산기'.
2. **제어장치 (Control Unit)**: 메모리에서 명령어를 읽어와서 해석하고, 하드웨어 장치들에 "지금 움직여!"라고 신호를 보내는 '지휘자'.
3. **레지스터 (Register)**: CPU가 당장 계산해야 할 데이터와 명령어 주소를 임시로 저장하는 '초고속 메모리'.

#### 🚌 버스 (Bus)
컴퓨터 내부 장치들이 서로 데이터를 주고받는 고속도로입니다. 메인보드에 회로로 깔려 있으며, 용도에 따라 세 종류로 나뉩니다.
* **데이터 버스**: 실제 데이터(명령어, 숫자 등)가 지나다니는 길.
* **주소 버스**: "어디로 보낼지" 메모리의 주소 정보가 지나다니는 길.
* **제어 버스**: CPU가 "읽기(Read)"를 할지 "쓰기(Write)"를 할지 제어 신호가 지나다니는 길.

#### 🔔 인터럽트 (Interrupt)
주변 장치가 CPU의 일을 방해하거나, 급한 일이 생겼음을 알리는 신호입니다. 
* 예시: CPU가 열심히 연산하고 있는데 사용자가 마우스를 움직이거나 키보드를 누르면, 입출력 장치가 CPU에 "나 지금 입력 들어왔어!"라고 인터럽트를 보냅니다.
* CPU는 하던 일을 잠시 레지스터/스택에 백업해 두고, **인터럽트 서비스 루틴(ISR)**을 실행해 급한 불을 끈 뒤 다시 본래 일로 돌아옵니다. 인터럽트 덕분에 컴퓨터가 실시간으로 주변 반응에 응답할수있습니다.

## 📂 Part 2. 메모리 (Memory)

---

### 1. 메모리 계층 (Memory Hierarchy)

컴퓨터 안에는 왜 이렇게 다양한 종류의 메모리가 필요할까요? 정답은 **'속도와 비용의 트레이드오프(Trade-off)'** 때문입니다. 빠르면 비싸고, 싸면 느립니다. 컴퓨터 설계자들은 이 문제를 해결하기 위해 메모리를 피라미드 구조로 쌓아 올렸습니다.

#### 💡 일상 비유로 이해하기
* **레지스터**: 지금 내 손에 들고 있는 펜
* **캐시 메모리**: 책상 위에 펼쳐둔 노트 (금방 집을 수 있음)
* **메인 메모리(RAM)**: 방 안의 책꽂이 (일어나서 걸어가야 함)
* **보조기억장치(HDD/SSD)**: 아파트 지하 창고 (엘리베이터 타고 한참 가야 함)

#### 🌲 메모리 계층 구조도
1. **레지스터 (Register)**: CPU 내부에 존재, 가장 빠름, 용량 극소(비트 단위).
2. **캐시 메모리 (Cache Memory - L1, L2, L3)**: CPU와 RAM 사이의 속도 차이를 줄여주는 버퍼. SRAM으로 구성.
3. **주기억장치 (Main Memory)**: 우리가 흔히 말하는 RAM. 컴퓨터가 켜져 있을 때 실행 중인 프로그램과 데이터가 올라옴. DRAM으로 구성.
4. **보조기억장치 (Secondary Storage)**: SSD, HDD. 전원이 꺼져도 데이터가 유지됨(비휘발성).

#### ⚡ 지역성의 원리 (Principle of Locality)
캐시 메모리가 '다음에 어떤 데이터가 필요할지' 예측하는 비결입니다. CPU가 자주 찾는 데이터는 정해져 있다는 법칙입니다.
* **시간 지역성 (Temporal Locality)**: 최근에 접근한 데이터에 곧 다시 접근할 확률이 높음 (예: 반복문의 루프 변수 `i`).
* **공간 지역성 (Spatial Locality)**: 최근에 접근한 데이터의 주변 데이터에 접근할 확률이 높음 (예: 배열의 0번 인덱스를 읽었다면 곧 1번, 2번 인덱스를 읽을 확률이 높음).

### 2. 메모리 관리 (Memory Management)

여러 프로그램이 동시에 실행되는 현대 컴퓨터에서는 제한된 RAM 자원을 공평하고 안전하게 나누어 쓰는 기술이 필수적입니다.

#### 🧠 가상 메모리 (Virtual Memory)
실제 물리적인 RAM 용량보다 더 큰 프로세스를 실행하기 위해, 보조기억장치(SSD)의 일부를 마치 RAM처럼 속여서 쓰는 기술입니다.
* **논리 주소(가상 주소)**: 프로그램 입장에서 "나한테 이만큼의 메모리가 있구나"라고 생각하는 주소 공간.
* **물리 주소**: 실제 RAM의 진짜 주소.
* **MMU (Memory Management Unit)**: CPU가 논리 주소를 줄 때마다, 이를 실제 물리 주소로 번역해 주는 하드웨어 장치.

#### 📄 메모리 할당 방식: 페이징과 세그멘테이션
가상 메모리를 실제 메모리에 어떻게 쪼개서 매핑할 것인가에 대한 방법입니다.

| 방식 | 설명 | 장점 | 단점 |
| :--- | :--- | :--- | :--- |
| **페이징 (Paging)** | 가상 메모리와 물리 메모리를 **같은 크기(Page/Frame)**로 뚝뚝 자르는 방식. | 구현이 쉽고 **외부 단편화**가 없음. | 프로세스 크기가 딱 안 맞으면 마지막 페이지에 공간이 남는 **내부 단편화** 발생. |
| **세그멘테이션 (Segmentation)** | 코드, 데이터, 스택 등 **의미나 기능 단위(Segment)**로 크기를 다르게 자르는 방식. | 공유와 보안(권한 부여) 측면에서 유리함. | 중간중간 애매한 빈 공간이 생겨 메모리가 낭비되는 **외부 단편화** 발생. |

*현대 운영체제는 주로 이 둘의 장점을 섞은 **'페이지화된 세그멘테이션'**을 사용합니다.*

#### ⚠️ 페이지 교체 알고리즘
RAM이 가득 찼을 때, 새로운 페이지를 올리기 위해 기존에 있던 페이지 중 어떤 것을 쫓아낼지 결정하는 규칙입니다.
* **FIFO (First-In-First-Out)**: 가장 먼저 들어온 페이지를 먼저 내보냄.
* **LRU (Least Recently Used)**: 가장 오랫동안 사용되지 않은 페이지를 내보냄 (가장 대중적이고 효율적).
* **LFU (Least Frequently Used)**: 방문 횟수가 가장 적은 페이지를 내보냄.
