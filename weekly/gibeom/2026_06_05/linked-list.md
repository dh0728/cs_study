# 연결리스트 (Linked List)

##  연결 리스트란?
데이터를 감싼 **노드**를 **포인터**로 연결해서 공간적인 효율성을 극대화시킨 자료구조

### 시간 복잡도
* **삽입 / 삭제:** $O(1)$
* **탐색:** $O(n)$

### 연결 리스트의 분류
1. **싱글 연결 리스트 (Singly Linked List):** 다음 노드만을 가리키는 포인터를 가짐
2. **이중 연결 리스트 (Doubly Linked List):** 이전 노드와 다음 노드를 가리키는 포인터를 가짐
3. **원형 이중 연결 리스트 (Circular Doubly Linked List):** 이중 연결 리스트와 같지만, 마지막 노드의 다음 포인터가 헤드 노드를 가리킴

###  장단점 요약
| 장점 | 단점 |
| :--- | :--- |
| **삽입/삭제가 빠르고 용이함 $O(1)$**<br>리스트의 길이가 가변적이며 따로 메모리 공간 할당 크기를 고정할 필요가 없어 삽입/삭제가 많은 연산에 유리함 | **자료 수정 및 탐색이 느림 $O(n)$**<br>특정 노드를 탐색하거나 데이터를 변경할 때 한 번에 접근할 수 없고, 무조건 연결 리스트를 처음부터 전부 순회(탐색)해야함 |

## 배열(Array)과 연결 리스트(Linked List) 비교

| 구분 | 배열 (Array) | 연결 리스트 (Linked List) |
| :--- | :--- | :--- |
| **장점** | 인덱스를 통한 빠른 직접 접근($O(1)$) 가능 | 삽입 및 삭제 연산($O(1)$)이 용이함 |
| **단점** | 중간 데이터의 삽입/삭제가 오래 걸림 ($O(n)$) | 랜덤 접근 불가능, 처음부터 탐색해야 함 ($O(n)$) |
| **용도** | 빠른 접근이 요구되고, 데이터 삽입/삭제가 적을 때 | 삽입/삭제 연산이 잦고, 데이터 검색 빈도가 적을 때 |
