# 시간복잡도와 공간복잡도

## 1. 복잡도 (Complexity)

좋은 알고리즘이란 **실행 시간도 짧으면서 저장 공간도 적게 쓰는 알고리즘** 그러나 일반적으로 시간과 공간은 서로 **반비례적인 경향**이 있어 둘 다 동시에 만족하기는 어려움 

최근에는 대용량 인프라와 하드웨어의 발전으로 시스템이 보편화되었기 때문에, 프로그램을 구현할 때 공간복잡도보다는 **시간복잡도가 우선순위**가 됨.

---

### 1.1 시간 복잡도 (Time Complexity)

####  시간 복잡도란?
> **"문제를 해결하는 데 걸리는 시간과 입력의 함수 관계"**

보통 명령문의 실행 빈도수에 따라 대략적인 소요 시간을 나타내기 위해 사용

#### 시간 복잡도가 필요한 이유
입력값이 적을 때는 알고리즘 간의 차이가 크지 않지만, **대용량 데이터(큰 입력값)**를 다룰 때 진가를 발휘. 입력값이 커질수록 연산 횟수가 기하급수적으로 늘어날 수 있기 때문에, 입력값이 증가함에 따라 증가하는 시간의 비율을 최소화한 효율적인 알고리즘을 구성해야 함.

####  Big-O(빅-오) 표기법이란?
입력 범위 $n$을 기준으로 로직이 몇 번 반복되는지 나타내는 표기법. **상한 점근**을 따르므로 **최악의 경우(Worst Case)**를 고려. 이를 통해 프로그램 실행 과정에서 발생할 수 있는 최악의 소요 시간까지 안전하게 고려가능.

* **예시:** $4n^2 + 3n + 21 \rightarrow O(n^2)$
  * 빅오 표기법은 가장 영향력이 큰 최고차항만을 남기고 계수와 나머지 항은 생략하여 표기

####  Big-O 표기법의 종류 및 특징

| 표기법 | 명칭 (시간) | 특징 | 대표 예시 / 자료구조 |
| :--- | :--- | :--- | :--- |
| **$O(1)$** | **상수 시간** | 입력의 크기와 상관없이 항상 같은 시간이 걸리는 알고리즘 | 배열(인덱스 참조), 해시 테이블 |
| **$O(\log n)$**| **로그 시간** | 문제를 해결하는 데 필요한 단계들이 연산마다 줄어드는 알고리즘 | 이진 탐색(Binary Search), 힙 삽입/삭제 |
| **$O(n)$** | **선형 시간** | 입력값이 증가함에 따라 시간도 같은 비율로 증가하는 알고리즘 | 정렬되지 않은 리스트에서 최댓값/최솟값 탐색 |
| **$O(n \log n)$**| **선형 로그 시간**| 데이터의 수가 2배로 늘 때, 연산 횟수는 2배보다 약간 더 많이 증가 | 힙 정렬(Heap Sort), 병합 정렬(Merge Sort) |
| **$O(n^2)$** | **제곱 시간** | 보통 2중 반복문(Nested Loop)을 도는 알고리즘 | 버블 정렬(Bubble Sort), 삽입 정렬 |
| **$O(2^n)$** | **지수 시간** | 입력값의 증가에 따라 연산 횟수가 제곱수 형태로 급격하게 증가 | 피보나치 수열(재귀 구현), 부분집합 구하기 |
| **$O(n!)$** | **팩토리얼 시간**| 빅오 표기법 중 가장 느린 속도, 데이터가 조금만 늘어도 처리 불가 | 순열(Permutation)의 모든 경우의 수 도출 |

####  시간 복잡도 속도 비교 (빠른 순 $\rightarrow$ 느린 순)
$$\small O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$$

### 1.2 공간 복잡도 (Space Complexity)

####  공간 복잡도란?
> **"프로그램을 실행시켰을 때 필요한 자원 공간(메모리)의 양"**

총 필요 저장 공간은 **고정 공간**과 **가변 공간**으로 구분

1. **고정 공간 (알고리즘과 무관한 공간):**
   * 프로그램 코드 저장 공간, 단순 변수 및 상수
2. **가변 공간 (알고리즘 실행 시 필요한 공간):**
   * 프로그램 실행 중 동적으로 할당되거나 필요로 하는 공간 (재귀 호출 스택, 동적 배열 등)

####  공간 복잡도 계산식
$$S(P) = c + S_p(n)$$

* $c$ : 고정 공간 (상수)
* $S_p(n)$ : 가변 공간

고정 공간은 항상 일정한 상수이므로, 실질적인 공간 복잡도는 **가변 공간($S_p(n)$)이 얼마나 필요한가**에 따라 결정
