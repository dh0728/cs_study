## 1. 알고리즘의 효율성 측정 기준: 복잡도(Complexity)

우리가 짠 코드가 얼마나 효율적인지 평가하기 위해 **시간복잡도**와 **공간복잡도**라는 기준을 사용합니다.

### ① 시간복잡도 (Time Complexity)

* **개념:** 컴퓨터의 사양이 제각각 다르기 때문에 단순 초(second) 단위로 측정하지 않고, "입력 데이터의 크기에 따라 핵심 로직이 몇 번 반복되는가"를 기준으로 측정합니다.

* **빅오 표기법 (Big-O Notation):** 수많은 연산 중 **가장 크게 영향을 미치는 핵심 항**만 남기고 나머지는 과감히 생략하여 효율성을 표현하는 방식입니다.

* *예시:* 연산 횟수가 $10n^2 + n$번인 알고리즘이 있다면, 데이터가 무한히 커질 때 $n^2$의 영향력이 압도적이므로 이를 $O(n^2)$로 표기합니다.

### ② 시간복잡도 속도 비교 그래프 (암기 필수!)

오른쪽으로 갈수록 데이터가 늘어날 때 컴퓨터가 과부하에 걸려 비효율적인 구조입니다.

$$\text{O(1)} < \text{O(logn)} < \text{O(n)} < \text{O(nlogn)} < \text{O(n}^2) < \text{O(2}^n) < \text{O(n!)}$$

* **$O(1)$ (상수 시간):** 데이터가 1개든 100만 개든 단 한 번에 값을 찾는 가장 이상적인 속도입니다 (예: 배열의 방 번호로 직접 찾기).

## 2. 데이터를 일렬로 줄 세우기: 선형 자료구조

데이터를 기차 칸처럼 일렬로 연속되게 나열하는 방식입니다.

### ① 정적 배열 (Array) vs 동적 배열 (Vector)

* **정적 배열 (Array):** 처음부터 "방을 딱 5칸만 쓰겠다!"라고 크기를 고정해 두는 상자입니다. 메모리 공간이 연속되어 있어 방 번호(인덱스)만 알면 단번에 찾아갈 수 있지만($O(1)$), 도중에 크기를 늘릴 수 없습니다.

* **동적 배열 (Vector):** 데이터가 늘어나면 컴퓨터가 알아서 상자 크기를 유연하게 늘려주는 스마트한 배열입니다.

### ② 연결 리스트 (Linked List)

* **개념:** 상자들이 메모리 여기저기에 흩어져 있고, 각 상자가 "다음 상자의 주소(이정표)"를 들고 연결되어 있는 형태입니다.

* **특징:** 기차 중간에 새로운 칸을 끼워 넣거나 빼는 것(삽입/삭제)은 이정표만 새로 연결하면 되니 빠릅니다. 하지만 특정 데이터를 찾으려면 맨 첫 칸부터 이정표를 따라 순서대로 걸어가야 하므로 탐색이 느립니다.

### ③ 스택 (Stack)과 큐 (Queue)

* **스택 (Stack):** "프링글스 과자통"을 생각하면 됩니다. 먼저 넣은 데이터가 바닥에 쌓이고, 가장 마지막에 넣은 데이터가 맨 위로 올라와 먼저 나옵니다 (LIFO - 후입선출).

* *활용:* 웹 브라우저의 '뒤로 가기', 실행 취소(Ctrl + Z)

* **큐 (Queue):** "선착순 줄서기"나 "터널"입니다. 먼저 들어간 데이터가 먼저 빠져나오는 정직한 구조입니다 (FIFO - 선입선출).

* *활용:* 프린터 인쇄 대기열, 은행 번호판 대기

## 3. 관계를 나타내는 구조: 비선형 자료구조

일렬이 아니라, 사방으로 뻗어 나가는 복잡한 관계를 표현합니다.

### ① 트리 (Tree)

* **개념:** 회사의 조직도나 가계도처럼 "계층 구조"를 가지는 뒤집힌 나무 모양의 구조입니다.

* **구성:** 맨 위의 대장 노드를 **루트(Root)**, 자식이 없는 말단 노드를 리프(Leaf)라고 부릅니다.

* **이진 탐색 트리 (BST):** 트리의 한 종류로, "나보다 작은 값은 왼쪽, 큰 값은 오른쪽"에 배치하여 원하는 값을 찾을 때마다 탐색 범위를 절반씩 뚝딱 깎아내어 매우 빠르게($O(\log n)$) 데이터를 찾게 해줍니다.

### ② 그래프 (Graph)

* **개념:** 지하철 노선도나 페이스북 친구 관계처럼, 점(정점)과 선(간선)으로 복잡하게 얽혀 있는 그물망 구조입니다. 지도 길 찾기나 네트워크 연결 상태를 표현할 때 주로 사용됩니다.

## 4. 번호표와 사물함: 해시 테이블 & 맵 (Hash Table & Map)

### ① 해시 테이블 (Hash Table)

* **개념:** 마법의 상자(해시 함수)에 '이름'을 집어넣으면 고유한 사물함 번호(숫자)를 즉시 뱉어내고, 그 사물함에 데이터를 보관하는 방식입니다.

* **장점:** 아무리 방대한 데이터가 있어도 이름표(Key)만 주면 사물함 번호로 바로 접근하므로 데이터를 찾는 속도가 압도적으로 빠릅니다 ($O(1)$).

## 💡 비전공자를 위한 스터디 팁 (비유로 기억하기)

* **배열**은 아파트입니다. 호수(방 번호)를 알면 바로 찾아가지만, 건물 크기를 갑자기 늘릴 순 없습니다.
* **연결 리스트**는 보물찾기입니다. 쪽지를 열면 다음 쪽지 위치가 적혀있어 순서대로 가야 하지만, 중간에 새 보물을 숨기기는 쉽습니다.
* **스택**은 밑이 막힌 상자(과자통)이고, **큐**는 양쪽이 뚫린 파이프(터널)입니다.
