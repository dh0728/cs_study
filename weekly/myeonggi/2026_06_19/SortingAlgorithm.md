# 3주차: 정렬 알고리즘

---

## 전체 관계

```
정렬 알고리즘 = 숫자를 순서대로 줄 세우기

기초 3개 (느림)          고급 3개 (빠름)
┌─────────────┐          ┌─────────────┐
│ 버블 정렬    │          │ 퀵 정렬      │
│ 선택 정렬    │          │ 합병 정렬    │
│ 삽입 정렬    │          │ 힙 정렬      │
└─────────────┘          └─────────────┘
  O(n²) 느림               O(n log n) 빠름
```

> O(n²) vs O(n log n): 데이터 100개 기준
> - O(n²) → 100 × 100 = **10,000번** 비교
> - O(n log n) → 100 × 7 = **700번** 비교

# 1. 기초 정렬 (느림)

## 1-1. 버블 정렬 (Bubble Sort)

### 한 줄 정의
> 옆에 있는 두 숫자 비교 → 왼쪽이 크면 바꿈 → 끝까지 반복

### 비유
> 물속 공기방울처럼 큰 수가 오른쪽 끝으로 둥둥 떠오름

### 동작 과정
```
[ 5, 3, 8, 1 ] 시작

1단계: 5>3 → 바꿈  →  [ 3, 5, 8, 1 ]
2단계: 5<8 → 그대로 → [ 3, 5, 8, 1 ]
3단계: 8>1 → 바꿈  →  [ 3, 5, 1, 8 ]  ← 8 맨 뒤 확정!

다시 처음부터 → 반복 → [ 1, 3, 5, 8 ] 완성
```

### 핵심 규칙
- 왼쪽 > 오른쪽 → **바꿈**
- 왼쪽 < 오른쪽 → **그대로**
- 1번 통과마다 → 제일 큰 수 맨 뒤 확정
- n번 통과 → 정렬 완성

### 파이썬 코드
```python
numbers = [5, 3, 8, 1, 4]

n = len(numbers)             # 숫자 총 개수

for i in range(n):           # 전체를 n번 통과
    for j in range(n-i-1):   # 이미 확정된 뒷부분 제외
        if numbers[j] > numbers[j+1]:        # 왼쪽이 더 크면
            numbers[j], numbers[j+1] = numbers[j+1], numbers[j]  # 자리 바꿈

print(numbers)  # [1, 3, 4, 5, 8]
```

## 1-2. 선택 정렬 (Selection Sort)

### 한 줄 정의
> 전체를 훑어서 제일 작은 거 찾기 → 맨 앞에 놓기 → 반복

### 비유
> 키 순서로 줄 세울 때, 제일 작은 사람 찾아서 맨 앞에 세우고, 남은 사람 중 또 찾기

### 동작 과정
```
[ 5, 3, 8, 1 ] 시작

1번째: 전체 훑음 → 1이 제일 작음 → 5랑 바꿈 → [ 1, 3, 8, 5 ]  ← 1 확정
2번째: [3,8,5] 훑음 → 3이 제일 작음 → 이미 제자리 → [ 1, 3, 8, 5 ]  ← 3 확정
3번째: [8,5] 훑음 → 5가 작음 → 8이랑 바꿈 → [ 1, 3, 5, 8 ]  ← 5 확정
4번째: [8] 혼자 → 자동 확정

완성: [ 1, 3, 5, 8 ]
```

### 핵심 규칙
- `min_idx` = 제일 작은 숫자의 위치를 기억하는 변수
- 매 단계마다 딱 **1번만 바꿈** (버블은 여러 번 바꿈)
- 앞에서부터 하나씩 확정

### 파이썬 코드
```python
numbers = [5, 3, 8, 1, 4]

n = len(numbers)

for i in range(n):
    min_idx = i                        # 현재 자리가 제일 작다고 가정
    for j in range(i+1, n):
        if numbers[j] < numbers[min_idx]:   # 더 작은 거 발견하면
            min_idx = j                      # 위치 기억
    numbers[i], numbers[min_idx] = numbers[min_idx], numbers[i]  # 맨 앞으로

print(numbers)  # [1, 3, 4, 5, 8]
```

### 버블과 차이

| | 버블 | 선택 |
|---|---|---|
| 바꾸는 횟수 | 많음 (이동마다) | 적음 (찾고 1번) |
| 확정 방향 | 뒤에서부터 | 앞에서부터 |

## 1-3. 삽입 정렬 (Insertion Sort)

### 한 줄 정의
> 카드 뽑기 → 왼쪽 정렬된 구역에서 맞는 자리 찾기 → 끼워 넣기

### 비유
> 카드게임에서 새 카드를 받을 때마다 손패의 맞는 위치에 끼워 넣는 것

### 동작 과정
```
[ 5, 3, 8, 1, 4 ] 시작
 ↑ 혼자라 정렬됨

3 뽑음 → 5보다 작음 → 5 앞에 삽입 → [ 3, 5, 8, 1, 4 ]
8 뽑음 → 5보다 큼   → 그대로       → [ 3, 5, 8, 1, 4 ]
1 뽑음 → 전부 밀기  → 맨 앞에 삽입 → [ 1, 3, 5, 8, 4 ]
4 뽑음 → 3 뒤, 5 앞 → 끼움         → [ 1, 3, 4, 5, 8 ]
```

### 핵심 규칙
- `key` = 지금 손에 든 카드
- 왼쪽 = **항상 정렬된 상태** 유지
- `while` = 작은 거 만날 때까지 왼쪽으로 이동
- 멈춘 바로 오른쪽 자리에 **꽂음**

### 파이썬 코드
```python
numbers = [5, 3, 8, 1, 4]

for i in range(1, len(numbers)):
    key = numbers[i]      # 카드 한 장 뽑기
    j = i - 1

    while j >= 0 and numbers[j] > key:   # 왼쪽에서 자리 찾기
        numbers[j+1] = numbers[j]         # 오른쪽으로 밀기
        j -= 1

    numbers[j+1] = key    # 찾은 자리에 삽입

print(numbers)  # [1, 3, 4, 5, 8]
```

# 2. 고급 정렬 (빠름)

## 2-1. 퀵 정렬 (Quick Sort)

### 한 줄 정의
> 기준점(피벗) 하나 잡기 → 작은 것 왼쪽, 큰 것 오른쪽 → 각 그룹에서 또 반복

### 비유
> 반 전체를 키 기준으로 "작은 팀 / 기준 / 큰 팀"으로 나눈 뒤, 각 팀 안에서 또 나누기

### 동작 과정
```
[ 5, 3, 8, 1, 4 ]  피벗 = 8 (중간값)

작은 것: [5, 3, 1, 4]  |  피벗: [8]  |  큰 것: []
           ↓ 또 퀵정렬
피벗 = 1
작은 것: []  |  [1]  |  큰 것: [5, 3, 4]
                             ↓ 또 퀵정렬
                        피벗 = 3
                        [] | [3] | [5, 4]
                                   ↓ 또 퀵정렬
                               [] | [4] | [5]

이어 붙이기: [1] + [3] + [4] + [5] + [8] = [1, 3, 4, 5, 8]
```

### 핵심 용어

| 용어 | 뜻 |
|---|---|
| 피벗 (pivot) | 기준점. 리스트 중간값 선택 |
| 재귀 (recursion) | 함수가 자기 자신을 다시 호출 |
| 탈출 조건 | 1개 이하면 멈춤 (없으면 무한 반복) |

### 파이썬 코드
```python
def quick_sort(arr):
    if len(arr) <= 1:             # 1개 이하 → 이미 정렬됨, 멈춤
        return arr

    pivot = arr[len(arr)//2]      # 중간값을 피벗으로
    left  = [x for x in arr if x < pivot]   # 작은 것들
    mid   = [x for x in arr if x == pivot]  # 같은 것들
    right = [x for x in arr if x > pivot]   # 큰 것들

    return quick_sort(left) + mid + quick_sort(right)  # 재귀 + 이어붙이기

print(quick_sort([5, 3, 8, 1, 4]))  # [1, 3, 4, 5, 8]
```

### 왜 빠른가?
- 피벗 하나로 리스트를 **반씩 줄임**
- 비교 횟수가 O(n²) → O(n log n)으로 확 줄어듦

## 2-2. 합병 정렬 (Merge Sort)

### 한 줄 정의
> 반으로 쪼개기 → 1개짜리 될 때까지 → 합치면서 정렬

### 비유
> 트럼프 카드 두 묶음을 각각 정렬해두고, 두 묶음 맨 위 카드를 비교해가며 합치기

### 동작 과정
```
쪼개기 (위→아래):
[ 5, 3, 8, 1 ]
    ↙        ↘
[ 5, 3 ]   [ 8, 1 ]
  ↙  ↘       ↙  ↘
[ 5 ][ 3 ] [ 8 ][ 1 ]   ← 1개짜리, 더 못 쪼갬

합치기 (아래→위):
[5]+[3] → 비교 → [3, 5]
[8]+[1] → 비교 → [1, 8]
[3,5]+[1,8] → 비교하며 합침 → [1, 3, 5, 8]
```

### 합치는 방법
```
[3, 5]  [1, 8]
 ↑       ↑
 3  vs  1  → 1 꺼냄  → result: [1]
 3  vs  8  → 3 꺼냄  → result: [1, 3]
 5  vs  8  → 5 꺼냄  → result: [1, 3, 5]
 남은 [8]  → 그냥 붙임 → result: [1, 3, 5, 8]
```

### 파이썬 코드
```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    mid   = len(arr) // 2
    left  = merge_sort(arr[:mid])   # 왼쪽 절반 재귀 정렬
    right = merge_sort(arr[mid:])   # 오른쪽 절반 재귀 정렬
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:       # 더 작은 거 먼저
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    return result + left[i:] + right[j:]  # 남은 것 붙이기

print(merge_sort([5, 3, 8, 1, 4]))  # [1, 3, 4, 5, 8]
```

### 퀵 정렬과 차이

| | 퀵 정렬 | 합병 정렬 |
|---|---|---|
| 방식 | 나누면서 정렬 | 합치면서 정렬 |
| 추가 공간 | 거의 없음 | 필요함 |
| 안정성 | 불안정 | 안정 |

## 2-3. 힙 정렬 (Heap Sort)

### 한 줄 정의
> 최대 힙 만들기 → 맨 위(최대값) 꺼내서 뒤로 → 힙 다시 정리 → 반복

### 비유
> 매번 제일 큰 사람 뽑아서 줄 맨 뒤에 세우기 반복 → 자연스럽게 오름차순 완성

### 전제 지식: 최대 힙

```
최대 힙 = 부모가 항상 자식보다 큰 트리

       8          ← 맨 위 = 항상 최대값
      / \
     5   3
    / \
   1   4
```

리스트로 표현:
- i번 노드의 왼쪽 자식 = `2*i + 1`
- i번 노드의 오른쪽 자식 = `2*i + 2`

### 동작 과정
```
[ 5, 3, 8, 1, 4 ] → 최대 힙 만들기 → [ 8, 4, 5, 1, 3 ]

1번: 맨 위 8 꺼냄 → 맨 뒤로 → [ 5, 4, 3, 1, | 8 ]  ← 8 확정
2번: 맨 위 5 꺼냄 → 맨 뒤로 → [ 4, 1, 3, | 5, 8 ]  ← 5 확정
3번: 맨 위 4 꺼냄 → 맨 뒤로 → [ 3, 1, | 4, 5, 8 ]  ← 4 확정
4번: 맨 위 3 꺼냄 → 맨 뒤로 → [ 1, | 3, 4, 5, 8 ]  ← 3 확정
완성: [ 1, 3, 4, 5, 8 ]
```

### 핵심 용어

| 용어 | 뜻 |
|---|---|
| heapify | 힙 규칙(부모≥자식)이 깨졌을 때 다시 맞추는 함수 |
| 최대 힙 | 부모가 항상 자식보다 큰 트리 |
| 루트 | 트리 맨 위. 최대 힙에서 항상 최대값 |

### 파이썬 코드
```python
def heapify(arr, n, i):
    largest = i
    left  = 2*i + 1    # 왼쪽 자식 인덱스
    right = 2*i + 2    # 오른쪽 자식 인덱스

    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right

    if largest != i:   # 자식이 더 크면 바꾸고 재귀
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)

def heap_sort(arr):
    n = len(arr)
    for i in range(n//2 - 1, -1, -1):   # 최대 힙 만들기
        heapify(arr, n, i)
    for i in range(n-1, 0, -1):          # 하나씩 꺼내기
        arr[0], arr[i] = arr[i], arr[0]  # 최대값 맨 뒤로
        heapify(arr, i, 0)               # 힙 다시 정리

numbers = [5, 3, 8, 1, 4]
heap_sort(numbers)
print(numbers)  # [1, 3, 4, 5, 8]
```

### 다른 정렬과 차이

| | |
|---|---|
| 버블/선택/삽입 | 앞에서부터 직접 비교 |
| 퀵/합병 | 나누거나 합치기 |
| 힙 | 트리 구조 활용해서 꺼내기 |
