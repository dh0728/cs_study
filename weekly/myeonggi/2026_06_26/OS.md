# 4주차: 운영체제 + 메모리

---

## 전체 관계

```
하드웨어 (CPU, RAM, SSD...)
    ↕  OS가 관리
운영체제 (OS)
    ↕  OS가 제공
앱 / 프로그램
```

> OS = 하드웨어와 앱 사이의 **중간 관리자**

---

# 1. 운영체제와 컴퓨터

## 1-1. 운영체제 (OS)란?

### 한 줄 정의
> 컴퓨터 전체를 관리하는 **총책임자**

### 비유
- 컴퓨터 = 식당
- OS = 식당 지배인
- CPU/메모리 = 주방/재료
- 앱 = 손님

손님(앱)이 주문하면 → 지배인(OS)이 주방(하드웨어)에 지시

### 실생활 예시

| OS | 쓰는 곳 |
|---|---|
| Windows | PC |
| macOS | 맥북 |
| Linux | 서버 |
| Android | 안드로이드폰 |
| iOS | 아이폰 |

---

## 1-2. 운영체제의 역할과 구조

### 한 줄 정의
> OS = **커널** + 그 위에 쌓인 것들

### 구조 (위에서 아래로)

```
[ 사용자 ]
    ↓
[ 앱 / 프로그램 ]
    ↓ 시스템 콜
[ 운영체제 (OS) ]
    [ 커널 (Kernel) ]  ← OS의 핵심
    [ 파일 관리 / 메모리 관리 / 프로세스 관리 ]
    ↓
[ 하드웨어 ]
```

### 핵심 용어

| 용어 | 뜻 | 비유 |
|---|---|---|
| 커널 (Kernel) | OS의 핵심 엔진 | 식당 주방장 |
| 시스템 콜 (System Call) | 앱이 OS한테 요청하는 방법 | 손님이 주문서 내는 것 |
| 프로세스 (Process) | 실행 중인 프로그램 | 주방에서 요리 중인 것 |

### OS의 4가지 역할

| 역할 | 설명 |
|---|---|
| 프로세스 관리 | 여러 앱이 동시에 돌아가도록 관리 |
| 메모리 관리 | 각 앱에게 메모리 나눠주고 회수 |
| 파일 관리 | 파일 저장, 읽기, 삭제 관리 |
| 입출력 관리 | 키보드, 마우스, 화면 등 제어 |

---

## 1-3. 컴퓨터의 요소

### 한 줄 정의
> 컴퓨터 = **CPU + 메모리 + 저장장치 + 입출력장치** → 버스로 연결

### 구조

```
[ 입력장치 ]  키보드, 마우스
      ↓
[ CPU ] ←→ [ 메모리 (RAM) ]
      ↓
[ 저장장치 ]  SSD, HDD
      ↓
[ 출력장치 ]  모니터, 스피커
```

모두 **버스(Bus)** 라는 통로로 연결됨

### 4가지 요소

| 요소 | 비유 | 역할 |
|---|---|---|
| CPU | 두뇌 | 계산하고 명령 처리 |
| 메모리 (RAM) | 책상 (작업대) | 지금 쓰는 것 올려놓는 곳 |
| 저장장치 (SSD/HDD) | 책장 (창고) | 전원 꺼져도 보관 |
| 입출력장치 | 손/눈/귀 | 사람이랑 소통 |

### 버스 (Bus)란?

> 각 요소가 데이터를 주고받는 **통로**

- 비유: 식당에서 요리 나르는 **서빙 통로**
- CPU ↔ 메모리 ↔ 저장장치가 버스로 연결

# 2. 메모리

## 2-1. 메모리 계층

### 한 줄 정의
> 빠를수록 비싸고 작음. 느릴수록 싸고 큼.

### 계층 구조 (위 = 빠름 / 아래 = 느림)

```
┌─────────────────────┐  ← 제일 빠름, 제일 작음, 제일 비쌈
│   레지스터 (Register) │  CPU 안에 있음. 바이트 단위
├─────────────────────┤
│   캐시 (Cache)       │  CPU 바로 옆. KB~MB 단위
├─────────────────────┤
│   RAM (주기억장치)    │  우리가 아는 메모리. GB 단위
├─────────────────────┤
│   SSD / HDD         │  ← 제일 느림, 제일 큼, 제일 쌈
└─────────────────────┘      (보조기억장치). TB 단위
```

### 비유: 요리사의 작업 공간

| 계층 | 비유 | 속도 | 용량 |
|---|---|---|---|
| 레지스터 | 요리사 손 안 | 즉각 | 아주 작음 |
| 캐시 | 조리대 위 | 매우 빠름 | 작음 |
| RAM | 주방 냉장고 | 빠름 | 중간 |
| SSD/HDD | 창고 | 느림 | 큼 |

### 왜 계층이 필요한가?
- 빠른 메모리는 너무 비쌈 → 전부 레지스터로 만들 수 없음
- 그래서 **자주 쓰는 것** = 위 계층 / **가끔 쓰는 것** = 아래 계층
- CPU가 데이터 필요할 때: 레지스터 → 캐시 → RAM → SSD 순서로 찾음

### 속도 비교 (대략)

| 계층 | 접근 속도 |
|---|---|
| 레지스터 | 1 나노초 이하 |
| 캐시 | 수 나노초 |
| RAM | 수십~수백 나노초 |
| SSD | 수십~수백 마이크로초 |
| HDD | 수 밀리초 |

## 2-2. 메모리 관리

### 한 줄 정의
> OS가 메모리를 **나눠주고, 추적하고, 회수**하는 것

### 왜 필요한가?
- 앱이 여러 개 동시에 실행됨
- 메모리 공간은 한정됨
- OS가 교통 정리를 해줘야 함

### 핵심 개념 3가지

#### 1) 가상 메모리 (Virtual Memory)

| | |
|---|---|
| 뭔지 | 실제 RAM이 부족할 때 SSD를 RAM처럼 쓰는 기술 |
| 비유 | 책상이 좁으면 바닥에 임시로 펼쳐두는 것 |
| 장점 | RAM보다 큰 프로그램도 실행 가능 |
| 단점 | SSD는 RAM보다 느림 → 속도 저하 |

#### 2) 페이징 (Paging)

| | |
|---|---|
| 뭔지 | 메모리를 **일정한 크기(페이지)** 로 잘라서 관리 |
| 비유 | 책을 페이지 단위로 나눠서 보관 |
| 장점 | 메모리 낭비 줄임. 조각조각 빈 곳 활용 가능 |

```
[ 메모리 ]
┌──────┐
│ 페이지1│ ← 앱A
├──────┤
│ 페이지2│ ← 앱B
├──────┤
│ 페이지3│ ← 앱A
├──────┤
│ 페이지4│ ← 비어있음
└──────┘
```

#### 3) 스와핑 (Swapping)

| | |
|---|---|
| 뭔지 | RAM이 꽉 차면 안 쓰는 것을 SSD로 내보내는 것 |
| 비유 | 책상이 꽉 차면 안 쓰는 책을 창고로 보내기 |
| 문제 | SSD ↔ RAM 이동이 느려서 컴퓨터가 버벅임 |

### 메모리 관련 오류

| 오류 | 설명 | 비유 |
|---|---|---|
| 메모리 누수 (Memory Leak) | 앱이 메모리 쓰고 반납 안 함 | 책상에서 책 빌려가고 안 돌려줌 |
| 페이지 폴트 (Page Fault) | 필요한 페이지가 RAM에 없어서 SSD에서 가져옴 | 책이 창고에 있어서 가지러 감 |

## 전체 요약

| 항목 | 핵심 한 줄 |
|---|---|
| 운영체제 | 하드웨어와 앱 사이 총책임자 |
| 역할과 구조 | 커널이 핵심. 프로세스/메모리/파일/입출력 관리 |
| 컴퓨터의 요소 | CPU(두뇌) + RAM(작업대) + SSD(창고) + 입출력장치 |
| 메모리 계층 | 빠를수록 비싸고 작음. 레지스터→캐시→RAM→SSD |
| 메모리 관리 | OS가 나눠주고 회수. 가상메모리/페이징/스와핑 |
