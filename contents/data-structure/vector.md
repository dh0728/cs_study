# 벡터 (Vector)

## 벡터란?

**동적으로 크기가 조절되는 배열**이다.

일반 배열은 선언 시 크기가 고정되지만, 벡터는 데이터가 추가될 때 자동으로 크기를 늘린다.

---

## 동적 크기 조절 방식

벡터는 내부적으로 고정 크기 배열을 가지고 있다. 데이터가 꽉 차면 더 큰 배열을 새로 할당하고 기존 데이터를 복사한다. 이때 보통 **현재 크기의 2배**로 늘린다.

```
[1 | 2 | 3 |   ]  ← 꽉 참
       ↓ 2배 확장
[1 | 2 | 3 |   |   |   |   |   ]  ← 새 배열로 복사
```

2배씩 늘리는 이유는 매번 확장하면 복사 비용이 크기 때문에, 여유 공간을 미리 확보해두기 위함이다.

---

## 파이썬에서의 벡터

파이썬의 **리스트(list)** 가 벡터와 동일하게 동작한다. 내부적으로 동적 배열로 구현되어 있다.

```python
arr = []

arr.append(1)   # [1]
arr.append(2)   # [1, 2]
arr.append(3)   # [1, 2, 3]

arr.pop()       # [1, 2]
```

---

## 주요 연산

| 연산 | 시간복잡도 | 설명 |
|------|-----------|------|
| 접근 | O(1) | 인덱스로 직접 접근 |
| 탐색 | O(n) | 순서대로 확인 |
| 맨 뒤 삽입 | O(1) | 평균적으로 상수 시간 |
| 중간 삽입/삭제 | O(n) | 이후 요소를 한 칸씩 이동 |

맨 뒤 삽입이 O(1)인 이유는 대부분의 경우 여유 공간에 바로 추가하면 되기 때문이다. 확장이 필요한 경우에만 O(n)이 발생하지만, 2배씩 늘리는 전략 덕분에 **평균적으로 O(1)** 로 취급한다.

## 배열과의 비교

| | 배열 | 벡터 |
|--|------|------|
| 크기 | 고정 | 동적 |
| 메모리 구조 | 연속 | 연속 |
| 접근 | O(1) | O(1) |
| 맨 뒤 삽입 | X | O(1) |

## 정리

- 벡터는 크기가 동적으로 조절되는 배열이다
- 꽉 차면 2배 크기의 새 배열을 할당하고 복사한다
- 파이썬 리스트가 벡터와 동일하게 동작한다
- 배열의 장점(빠른 접근)을 유지하면서 크기 제한을 해결한 자료구조다
