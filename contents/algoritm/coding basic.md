## 진법

컴퓨터는 전기 신호의 on/off, 즉 **0과 1만 이해**한다. 그래서 모든 데이터는 내부적으로 2진수로 표현된다.

| 진법 | 사용 숫자 | 표기 접두사 | 예시 (10진수 13) |
| --- | --- | --- | --- |
| 2진법 (Binary) | 0~1 | `0b` | `0b1101` |
| 10진법 (Decimal) | 0~9 | 없음 | `13` |
| 16진법 (Hexadecimal) | 0~9, A~F | `0x` | `0xD` |

```
10진수 13을 각 진법으로:

2진수 :  1101   (8 + 4 + 0 + 1)
16진수:  D      (13 = D)
```

> 💡 **16진법을 쓰는 이유?**
> 2진수는 자릿수가 너무 길어진다. 16진수 1자리가 2진수 4자리와 정확히 대응되기 때문에, 긴 2진수를 짧게 표현하는 용도로 자주 쓰인다. (색상 코드 `#FF0000`, 메모리 주소 `0x7fff` 등)

## 논리회로

**0과 1** 두 신호를 입력받아 정해진 규칙대로 하나의 출력을 내보내는 회로다. 기본 단위를 **게이트(Gate)** 라고 한다.

각 게이트의 동작은 **진리표(Truth Table)** 로 표현한다. 입력에 따른 출력을 모두 나열한 표다.

| 게이트 | 의미 | 입력 0,0 | 0,1 | 1,0 | 1,1 |
| --- | --- | --- | --- | --- | --- |
| **AND** | 둘 다 1일 때만 1 | 0 | 0 | 0 | 1 |
| **OR** | 하나라도 1이면 1 | 0 | 1 | 1 | 1 |
| **XOR** | 두 입력이 다를 때만 1 | 0 | 1 | 1 | 0 |
| **NAND** | AND의 반대 | 1 | 1 | 1 | 0 |
| **NOR** | OR의 반대 | 1 | 0 | 0 | 0 |

| 게이트 | 의미 | 입력 0 | 입력 1 |
| --- | --- | --- | --- |
| **NOT** | 입력을 반전 | 1 | 0 |

> 💡 AND·OR·NOT 세 가지만 있으면 어떤 논리회로든 만들 수 있다. 컴퓨터의 모든 연산은 결국 이 게이트들의 조합이다.

## 불대수

논리회로를 **수식으로 다루는 수학**이다. 참(1)과 거짓(0) 두 값만 사용하며, 프로그래밍의 조건문(`if`)과 직접 연결된다.

### 기본 연산

| 불대수 | 기호 | 프로그래밍 | 의미 |
| --- | --- | --- | --- |
| AND | $A \cdot B$ | `and` / `&&` | 논리곱 |
| OR | $A + B$ | `or` / `\|\|` | 논리합 |
| NOT | $A'$ | `not` / `!` | 부정 |

### 기본 법칙

```
교환법칙   A · B = B · A          A + B = B + A
결합법칙   (A · B) · C = A · (B · C)
분배법칙   A · (B + C) = A·B + A·C
항등법칙   A · 1 = A              A + 0 = A
```

### 드모르간의 법칙 (De Morgan's Law)

가장 자주 쓰이는 법칙이다. **부정을 분배할 때 AND와 OR가 뒤바뀐다.**

```
(A · B)' = A' + B'
(A + B)' = A' · B'
```

프로그래밍에서 조건을 뒤집을 때 그대로 적용된다.

```python
# 아래 두 줄은 완전히 동일하다
not (a and b)
(not a) or (not b)
```

> 💡 복잡한 `if` 조건을 단순하게 정리하거나, 부정 조건을 다룰 때 드모르간의 법칙을 알면 코드가 훨씬 깔끔해진다.

## 슈도코드

특정 프로그래밍 언어의 문법에 얽매이지 않고, **알고리즘의 흐름을 사람이 읽기 쉬운 형태로 표현**한 코드다.

- 실제로 실행되는 코드가 아니다
- 언어를 몰라도 로직을 이해할 수 있다
- 알고리즘 설계 단계에서 아이디어를 정리할 때 쓴다

### 작성 관례

- 들여쓰기로 코드 블록(범위)을 구분한다
- `if`, `while`, `for`, `return` 같은 핵심 키워드는 그대로 쓴다
- 세부 문법(세미콜론, 자료형 선언 등)은 생략한다

### 예시

최대값 찾기
```
findMax(arr):
    max = arr[0]
    for each x in arr:
        if x > max:
            max = x
    return max
```
우유 1개 사와, 계란 있으면 6개 사와
```
장보기():
    buy_milk(1)
    if 계란있으면
        buy_egg(6)
```
서비스 흐름
```
사용자 입력 받기
데이터 검증
검증 성공 시 저장, 실패 시 재입력
```

> 💡 슈도코드에는 정해진 표준이 없다. 읽는 사람이 흐름을 이해할 수 있으면 충분하다.

## 자주 쓰이는 약어

변수명이나 함수명에서 자주 쓰이는 줄임말이다. 코드를 읽다 보면 반복적으로 등장하므로 알아두면 의미 파악이 빨라진다.

### 값·임시 데이터

| 약어 | 원어 | 의미 |
| --- | --- | --- |
| **tmp** / **temp** | temporary | 임시 값 |
| **val** | value | 값 |
| **var** | variable | 변수 |
| **arg** | argument | 인자 |
| **param** | parameter | 매개변수 |
| **res** | result / response | 결과 / 응답 |
| **ret** | return | 반환값 |

### 반복·인덱스

| 약어 | 원어 | 의미 |
| --- | --- | --- |
| **i, j, k** | index | 반복문 인덱스 (관례적으로 i부터 사용) |
| **idx** | index | 인덱스 |
| **cnt** | count | 개수 |
| **num** | number | 숫자 |
| **len** | length | 길이 |

### 동작·상태

| 약어 | 원어 | 의미 |
| --- | --- | --- |
| **ck** / **chk** | check | 확인 |
| **init** | initialize | 초기화 |
| **calc** | calculate | 계산 |
| **func** / **fn** | function | 함수 |
| **prev** | previous | 이전 |
| **cur** / **curr** | current | 현재 |
| **flag** | flag | 상태 표시용 참/거짓 값 |

### 자료·구조

| 약어 | 원어 | 의미 |
| --- | --- | --- |
| **arr** | array | 배열 |
| **str** | string | 문자열 |
| **char** | character | 문자 |
| **obj** | object | 객체 |
| **msg** | message | 메시지 |
| **err** | error | 에러 |

> 💡 줄임말은 편하지만 과하면 오히려 읽기 어려워진다. `tmp`, `idx`, `cnt`처럼 널리 통용되는 것 위주로 쓰고, 의미가 모호할 수 있는 변수는 풀어 쓰는 것이 좋다.

## 상대경로와 절대경로

파일이나 폴더의 **위치를 표현하는 방식**이다. 두 가지가 있다.

### 절대경로 (Absolute Path)

- **최상위(루트)부터 시작하는 전체 경로**
- 어디서 실행하든 항상 같은 위치를 가리킨다

```
/Users/jiwon/project/data/file.txt      (macOS/Linux)
C:\Users\jiwon\project\data\file.txt     (Windows)
```

### 상대경로 (Relative Path)

- **현재 위치를 기준으로 표현하는 경로**
- 현재 어디에 있느냐에 따라 가리키는 대상이 달라진다

| 기호 | 의미 |
| --- | --- |
| `.` | 현재 폴더 |
| `..` | 상위 폴더 (한 단계 위) |
| `/` | 경로 구분자 |

```
현재 위치: /Users/jiwon/project

./data/file.txt      → /Users/jiwon/project/data/file.txt
../file.txt          → /Users/jiwon/file.txt   (한 단계 위로)
../../file.txt       → /Users/file.txt         (두 단계 위로)
```
